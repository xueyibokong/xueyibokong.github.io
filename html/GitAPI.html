• GIT
  • 工作区： Workspace
    暂存区： Index / Stage
    仓库区（或本地仓库）： Repository
    远程仓库： Remote
    命令
    作用
    注意
    旁白
  • GIT 初始化
  • 命令： git init
    作用： 在当前目录新建一个Git代码库
  • 命令： git init [project-name]
    作用： 新建一个目录，将其初始化为Git代码库
  • GIT 配置
  • 命令： git config --list
    作用： 显示当前的Git配置
  • 命令： git config -e [--global]
    作用： 编辑Git配置文件
  • 命令： $ git config [--global] user.name "[name]"
    $ git config [--global] user.email "[email address]"
    作用： 设置提交代码时的用户信息
  • GIT 克隆、GIT 关联
  • 命令： git clone [url]
    作用： 下载一个项目和它的整个代码历史
    注意： 本地没有仓库clone只会把远端仓库的master分支clone下来
  • 命令： git remote add origin [url]
    作用： 将本地仓库关联到远程仓库
  • GIT 文件（增加/删除）
  • 命令： git add [file1] [file2] ...
    作用： 添加指定文件到暂存区
  • 命令： git add [dir]
    作用： 添加指定目录到暂存区，包括子目录
  • 命令： git add .
    作用： 添加当前目录的所有文件到暂存区
  • 命令： git add -p
    作用： 添加每个变化前，都会要求确认; 对于同一个文件的多处变化，可以实现分次提交
  • 命令： git rm [file1] [file2] ...
    作用： 删除工作区文件，并且将这次删除放入暂存区
  • 命令： git rm --f [file1] [file2] ...
    作用： 不仅将该文件从缓存中删除，还会将物理文件删除（不会回收到垃圾桶）
  • 命令： git clean -df
    作用： 清理本地缓存
  • 命令： git rm --cached [file]
    作用： 停止追踪指定文件，但该文件会保留在工作区（即从缓存区删除）
  • 命令： git mv [file-original] [file-renamed]
    作用： 重命名文件，并且将这个重命名放入暂存区
  • GIT 提交
  • 命令： git commit -m [message]
    作用： 提交暂存区到仓库区
    注意： [message]中内容必须写
  • 命令： git commit [file1] [file2] ... -m [message]
    作用： 提交暂存区的指定文件到仓库区
  • 命令： git commit -a
    作用： 提交工作区自上次commit之后的变化，直接到仓库区
    注意： 这个不错，适合短时间内多次提交（但是要对自己工作区变化比较清楚）
  • 命令： git commit -v
    作用： 提交时显示所有diff信息
  • 命令： git commit --amend -m [message]
    作用： 使用一次新的commit，替代上一次提交；如果代码没有任何新变化，则用来改写上一次commit的提交信息
  • 命令： git commit --amend [file1] [file2] ...
    作用： 重做上一次commit，并包括指定文件的新变化
  • GIT 分支
  • 命令： git branch
    作用： 列出所有本地分支
  • 命令： git branch -r
    作用： 列出所有远程分支
  • 命令： git branch -a
    作用： 列出所有本地分支和远程分支
  • 命令： git branch [branch-name]
    作用： 新建一个分支，但依然停留在当前分支
  • 命令： git checkout -b [branch]
    作用： 新建一个分支，并切换到该分支
  • 命令： git branch [branch] [commit]
    作用： 新建一个分支，指向指定commit
  • 命令： git branch --track [branch] [remote-branch]
    作用： 新建一个分支，与指定的远程分支建立追踪关系
  • 命令： git checkout -b mybranch origin/mybranch
    作用： 取远程分支并分化一个新分支
  • 命令： git checkout -
    作用： 切换到上一个分支
  • 命令： git branch --set-upstream [branch] [remote-branch]
    作用： 建立追踪关系，在现有分支与指定的远程分支之间
  • 命令： git merge [branch]
    作用： 合并指定分支到当前分支
  • 命令： git cherry-pick [commit]
    作用： 选择一个commit，合并进当前分支
  • 命令： git branch -d [branch-name]
    作用： 删除分支
  • 命令： git branch -d [branch-name]
    作用： 强制删除分支
  • 命令： git push origin --delete [branch-name]
    git branch -dr [remote/branch]
    作用： 删除远程分支
  • 命令： git push origin :[branch-name]
    作用： 推送空分支到远程分支（删除远程分支的另一种做法）
  • GIT 标签
  • 命令： git tag
    作用： 列出所有tag
  • 命令： git tag [tag]
    作用： 新建一个tag在当前commit
  • 命令： git tag -d [tag]
    作用： 删除本地tag
  • 命令： git push origin :refs/tags/[tagName]
    作用： 删除远程tag
    注意： 创建空tag并push到远端（即删除远端tag）
  • 命令： git show [tag]
    作用： 查看tag信息
  • 命令： git push [remote] [tag]
    作用： 提交指定tag
  • 命令： git push [remote] --tags
    作用： 提交所有tag
  • 命令： git checkout -b [branch] [tag]
    作用： 新建一个分支，指向某个tag
  • GIT 查看
  • 命令： git status
    作用： 命令可以列出当前目录所有还没有被git管理的文件和被git管理且被修改但还未提交(git commit)的文件
  • 命令： git log
    作用： 显示当前分支的版本历史
  • 命令： git log --stat
    作用： 显示commit历史，以及每次commit发生变更的文件
  • 命令： git log -S [keyword]
    作用： 搜索提交历史，根据关键词
  • 命令： git log [tag] HEAD --pretty=format:%s
    作用： 显示某个commit之后的所有变动，每个commit占据一行
  • 命令： git log [tag] HEAD --grep feature
    作用： 显示某个commit之后的所有变动，其"提交说明"必须符合搜索条件
  • 命令： git log --follow [file]
    $ git whatchanged [file]
    作用： 显示某个文件的版本历史，包括文件改名
  • 命令： git log -p [file]
    作用： 显示指定文件相关的每一次diff
  • 命令： git log -5 --pretty --oneline
    作用： 显示过去5次提交
    注意： 5 可以换成其他数字
  • 命令： git shortlog -sn
    作用： 显示所有提交过的用户，按提交次数排序
  • 命令： git blame [file]
    作用： 显示指定文件是什么人在什么时间修改过
  • 命令： git diff
    作用： 显示暂存区和工作区的差异
    注意： 常用
  • 命令： git diff --cached [file]
    作用： 显示暂存区和上一个commit的差异
  • 命令： git diff HEAD
    作用： 显示工作区与当前分支最新commit之间的差异
  • 命令： git diff [first-branch]...[second-branch]
    作用： 显示两次提交之间的差异
  • 命令： git diff --shortstat "@{0 day ago}"
    作用： 显示今天你写了多少行代码
  • 命令： git show [commit]
    作用： 显示某次提交的元数据和内容变化
  • 命令： git show --name-only [commit]
    作用： 显示某次提交发生变化的文件
  • 命令： git show [commit]:[filename]
    作用： 显示某次提交时，某个文件的内容
  • 命令： git reflog
    作用： 显示当前分支的最近几次提交
  • GIT 远端同步
  • 命令： git fetch [remote]
    作用： 下载远程仓库的所有变动
  • 命令： git fetch -p
    作用： 取回远程仓库的变化,且删除本地存在而在远端已删除的分支
  • 命令： git remote -v
    作用： 显示所有远程仓库
    注意： 可以看到对远端仓库的权限（pull）（push）
  • 命令： git remote show [remote]
    作用： 显示某个远程仓库的信息
  • 命令： git remote add [shortname] [url]
    作用： 增加一个新的远程仓库，并命名
  • 命令： git pull [remote] [branch]
    作用： 取回远程仓库的变化，并与本地分支合并
  • 命令： git pull -p
    作用： 取回远程仓库的变化，并与本地分支合并,且删除本地存在而在远端已删除的分支
  • 命令： git push [remote] [branch]
    作用： 上传本地指定分支到远程仓库
  • 命令： git push [remote] --force
    作用： 强行推送当前分支到远程仓库，即使有冲突
  • 命令： git push [remote] --all
    作用： 推送所有分支到远程仓库
  • 命令： git push -f [remote] --all
    作用： 强制推送所有分支到远程仓库
  • GIT 撤销（版本回退）
  • 命令： git checkout [file]
    作用： 恢复暂存区的指定文件到工作区
    注意： 【常用】
  • 命令： git checkout [commit] [file]
    作用： 恢复某个commit的指定文件到暂存区和工作区
  • 命令： git checkout .
    作用： 恢复暂存区的所有文件到工作区
  • 命令： git reset [file]
    作用： 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变
  • 命令： git reset --hard
    作用： 重置暂存区与工作区，与上一次commit保持一致
  • 命令： git reset [commit]
    作用： 重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变
  • 命令： git reset --hard [commit]
    作用： 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致
  • 命令： git reset --keep [commit]
    作用： 重置当前HEAD为指定commit，但保持暂存区和工作区不变
  • 命令： git revert [commit]
    作用： 新建一个commit，用来撤销指定commit;后者的所有变化都将被前者抵消，并且应用到当前分支
  • 命令： git stash
    git stash pop
    作用： 暂时将未提交的变化移除，稍后再移入
  • GIT 打包
  • 命令： git archive
    作用： 生成一个可供发布的压缩包
  • GIT 问题解决
  • 冲突解决： 在合并分支时出现冲突后，首先将冲突文件手动解决，然后将解决后的文件添加到暂存区， 然后commit，然后push到远端。
    注意： 在push时可能会报错![rejected]，这是因为远端分支上又有了新的commit，此时需先执行 git pull [remotename] [branchname]，然后再push。
  • 命令： git config http.postBuffer 524288000
    作用： 修改git的传输字节限制。
    注意： 在https协议下做push时有时会遇到报错为： error: RPC failed; HTTP 411 curl 22 The requested URL returned error: 411 Length Required
